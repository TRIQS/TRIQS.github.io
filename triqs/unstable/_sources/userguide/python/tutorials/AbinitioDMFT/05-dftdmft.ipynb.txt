    "We saw previously in the lectures that quantum embedding boils down to upfolding the self-energy that we compute from a quantum impurity problem back to the global electronic structure of a crystal. This means the lattice self-energy is obtained from\n",
    "\n",
    "$$ \\Sigma_{\\nu\\nu'}^{\\sigma}(\\mathbf{k},\\omega) = [P^{\\sigma}_{m\\nu}(\\mathbf{k})]^{\\dagger}[\\Sigma_{\\mathrm{embed}}(\\omega)]_{m m'}^{\\sigma} P^{\\sigma}_{m'\\nu'}(\\mathbf{k}), $$\n",
    "where $m$ are orbital indices and $\\nu$ are Kohn-Sham band indices. We refer to $P^{\\sigma}_{m\\nu}(\\mathbf{k})$ as a projector which projects the Kohn-Sham states onto a set of localized atomic orbitals. In practice, in order to connect to electronic structure codes like density-functional theory or many-body perturbation theory, we need the Kohn-Sham band structure ($\\varepsilon^{\\sigma}_{\\nu}(\\mathbf{k})$) and the projector $P^{\\sigma}_{m\\nu}(\\mathbf{k})$. The TRIQS ecosystem supports interfaces (converters) to many different electronic structure codes available to the community. The currently supported DFT codes that have interfaces that support charge self-consistent DFT + DMFT are listed below in the table. The projection scheme corresponds to the program that is used to create the projectors mentioned above.\n",
    "\n",
    "| DFT Code |  Basis | Projection Scheme(s) |\n",
    "|:----------|:--------|:--------------------:|\n",
    "| [Wien2k](http://susi.theochem.tuwien.ac.at) |  All-electron, full potential (LAPW+lo) | [Projectors (dmftproj)](https://triqs.github.io/dft_tools/latest/guide/conv_wien2k.html)  |\n",
    "| [VASP](https://www.vasp.at)    | Pseudopotentials (PAW) | [Projectors (LOCPROJ)](https://triqs.github.io/dft_tools/latest/guide/conv_vasp.html), [Wannier90](https://triqs.github.io/dft_tools/latest/guide/conv_W90.html) |\n",
    "| [Quantum Espresso](https://www.quantum-espresso.org) | Pseudopotentials | [Wannier90](https://triqs.github.io/dft_tools/latest/guide/conv_W90.html) |\n",
    "| [Elk](https://elk.sourceforge.io) | All-electron, full potential (LAPW) | [Projectors](https://triqs.github.io/dft_tools/latest/guide/conv_elk.html) |\n",
    "\n",
    "The tutorials for SrVO$_{3}$ and La$_{2}$CuO$_{4}$ were produced using the [Wien2k DFT code](), which is an all-electron, full potential code and often referred to as the gold standard for total energy calculations. TRIQS provides a Fortran90 program ``dmftproj`` which creates the projectors from the Wien2k data. We can then use the ``Wien2kConverter`` from our suite of DFT converters to prepare an HDF5 file which contains all of the data that we need to run a DMFT calculation. In the following, we describe the input files for ``dmftproj`` for SrVO$_{3}$ and La$_{2}$CuO$_{4}$. Additionally, we provide a utility program called ``init_dmftpr`` which helps the user prepare the input file for your material. For more information and details on how the DFT calculation was run and how the projectors were created, see the remainder of the notebook."

    "While we are demonstrating the workflow for Wien2k + TRIQS, the general workflow is the same for all of the DFT codes that TRIQS has an interface to.     "\n",
    "- [Wannier90Converter](https://triqs.github.io/dft_tools/latest/_python_api/triqs_dft_tools.converters.wannier90.Wannier90Converter.html)\n",
    "- [Wien2kConverter](https://triqs.github.io/dft_tools/latest/_python_api/triqs_dft_tools.converters.wien2k.Wien2kConverter.html#triqs_dft_tools.converters.wien2k.Wien2kConverter)\n",
    "- [VaspConverter](https://triqs.github.io/dft_tools/latest/_python_api/triqs_dft_tools.converters.vasp.VaspConverter.html#triqs_dft_tools.converters.vasp.VaspConverter)\n",

    "```\n",

    "This will produce a ``case.h5`` file which contains the group ``dft_input``."

    "##### DFT\n",
    "In Wien2k, we can set up all the input files from only the Wien2k structure file which we provide in reference data. To follow along, one can execute the following commands:\n",
    "\n",
    "**Initialization**\n",
    "```bash\n",
    "init_lapw -prec 1 -vxc LDA\n",
    "```\n",
    "\n",
    "**Self-consistent field**\n",
    "```bash\n",
    "run_lapw -ec 0.00001 -cc 0.0001\n",
    "```\n",
    "\n",
    "For more details, we refer to the Wien2k documentation.\n",
    "\n",
    "##### Projectors\n",
    "\n",
    "To create the projectors, we need to run the following commands\n",
    "```bash\n",
    "x lapw1       # compute the KS eigenvalues\n",
    "x lapw2 -almd # compute coefficients in the local orbital basis\n",
    "```\n",
    "\n",
    "The ``dmftproj`` program takes the following input file which can be generated using ``init_dmftpr``:\n",
    "\n",
    "```\n",
    "!case.indmft\n",
    "4         ! number of atoms in the structure file\n",
    "2 1 2 2   ! multiplicity of each atom\n",
    "3         ! the maximum angular quantum number l; we repeat the following block for each atom\n",
    "cubic     ! (La) choice of spherical harmonics (cubic/spherical/custom)\n",
    "0 0 1 0   ! Create a projector for s/p/d/f? 0 = No, 1 = Yes (do not treat as correlated), 2 = Yes (treat as correlated)\n",
    "0 0 0 0   ! Split into irreps 0 = No, X = number of irreps\n",
    "cubic     ! (Cu) cubic harmonics\n",
    "0 0 2 0   ! Treat d-orbitals as correlated\n",
    "0 0 0 0   ! No splitting\n",
    "0         ! Do we have spin-orbit? 0 = No, 1 = Yes\n",
    "cubic     ! (O) cubic harmonics\n",
    "0 1 0 0   ! Create a projector for p\n",
    "0 0 0 0   ! No splitting\n",
    "cubic     ! (O) cubic harmonics\n",
    "0 1 0 0   ! Create a projector for p\n",
    "0 0 0 0   ! No splitting\n",
    "-0.73 0.73 ! energy window around the Fermi level (in Ry)\n",
    " ```"
