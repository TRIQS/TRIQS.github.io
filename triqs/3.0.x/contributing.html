Contributing¶

Note

Do not open an issue on github if you have questions regarding the usage of the code which are specific to your problem. Consolidate the Documentation and the userguide first. Should this not answer your questions, you can get in contact with other users and developers via our Google group.

Issues and feature requests¶

Please report bugs and feature request directly on the github issue page. In order to make it easier for us to treat your issue please use and fill one of the templates provided to you when opening a new issue.

Contributing to code and documentation¶

We are happy to receive your improvements and extensions to the code and documentation as pull requests on github. Please coordinate the implementation of new features and major changes directly with the TRIQS developers.

  • How to make a pull request?
  • How to contribute to documentation?

Google group / mailing list¶

As a platform for discussions with other users and developers we have set up a Google group. This channel is also used for the announcement of new releases and code updates.
