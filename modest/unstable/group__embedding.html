Embedding

Detailed Description

The embedding abstraction in ModEST. The embedding class describes a mapping between the impurity solvers and the the local correlated space ùìí. We provide factory functions to construct the embedding from the Local space within a One-body elements. The embedding class has operations (Emebdding operations) that allow the user to modify/describe their embedding task and methods (Emebdding methods) that act on Green's functions and self-energies.

The embedded self-energy is constructed from \(n_{\mathrm{imp}}\) impurity model self-energies. We denote by \([\Sigma_{\mathrm{imp}}]^{n_{\mathrm{imp}}, \tau}\) the self-energy of the impurity model \(n_{\mathrm{imp}}\), with block index \(\tau\). The embedding is given by :

$$ [\Sigma_{\mathrm{embed}}]^{\sigma}_{(\alpha, m_{\alpha}), (\alpha, m_{\alpha}')} = [\Sigma_{\mathrm{imp}}]^{n_{\mathrm{imp}(\alpha)},\tau(\sigma)}_{(\gamma(\alpha), m_{\alpha}),(\gamma(\alpha), m_{\alpha}')}.$$

In simple words, the self-energy of the block \(\alpha, \sigma\) is given by impurity self-energy of the impurity \(n_{\mathrm{imp}}(\alpha)\), block \(\gamma(\alpha),\tau(\sigma)\). This naturally requires that the blocks \(\alpha\) (embedding) and \(\gamma\) (impurity) have the same dimensions. The embedding is therefore completely characterized by a function : $$\psi (\alpha, \sigma) \rightarrow (n_\mathrm{imp}, \gamma, \tau)$$

This covers the following cases:

  • Equivalent atoms: use the same impurity model (i.e. same \(n_{\mathrm{imp}}\)) for different blocks \(\alpha\).
  • Magnetic order: just reverse the spin for some atoms, \(\tau(\sigma) = 1-\sigma\)
  • Splitting impurity models : Depending on the block structure of the problem, the corresponding impurity model connected to atom $a$ can be split. Different impurity solvers can then be used for each of these split impurity models. The self-energies from both models map back to the a single atom $a$. Or a trivial solver is used to obtain zero self-energy (drop).
  • cluster DMFT: re-group a subset of atoms $a$ into a larger `‘super atom’' which will be treated as a cluster DMFT problem.

Note that all the rotations (e.g. local to global coordinate systems) have been absorbed in the projector $P$ during the basis preparation, which makes the embedding itself very simple. We first prepare the proper basis for the embedding, and then perform the embedding.

Modules

 Emebdding factories
 Embedding makers which allow you to construct different embeddings tailored to your embedding scenario. Most of the makers require the Local space plus some additional information regarding how to decompose that local space into impurities.
 
 Emebdding methods
 The embedding methods are functions that can be used to manipulate the block structures of matrices, tensors, or TRIQS Green's functions. The two methods are: (1) embed and (2) extact.
 
 Emebdding operations
 The embedding operations are your interface to adapt and modify any embedding at will. Each operation returns a new Embedding, so you can string ops together or apply them iteratively. The embedding object is interactive, so we encourage "playing" with the ops to modify it's structure until your happy with the embedding description.
